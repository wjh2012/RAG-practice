# Step 0: Naive RAG의 한계

## 한계 3가지

### 1. 청킹 문제: 글자 수로 자르니까 문장이 중간에 잘림

```
"이란 반군 중 가장 강한 쿠르드족이 참" | "전하여 수천명 규모 지상전을 시작했다"
```

의미가 깨진 청크 = 검색 품질 저하

### 2. 검색 문제: 질문과 "단어"가 비슷한 청크만 찾음

```
질문: "중동 분쟁의 경제적 영향은?"
→ "유가 상승", "LNG 생산 차질" 같은 간접 관련 청크를 못 찾을 수 있음
```

### 3. 순위 문제: 벡터 유사도 Top 3가 진짜 최고인지 보장 없음

관련 없는 청크가 섞여 들어오면 답변 품질 하락

## Advanced RAG 해결 기법

| 한계 | 해결 기법 | 핵심 |
|---|---|---|
| 청킹이 의미를 깨뜨림 | **Semantic Chunking** | 의미 단위로 자르기 |
| 질문 표현이 제한적 | **Query Transformation** | 질문을 변환/확장 |
| 검색 결과 품질 불안정 | **Re-ranking** | 검색 후 재정렬 |
