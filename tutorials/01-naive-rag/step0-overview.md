# Step 0: RAG 큰 그림 이해

## RAG란?

**R**etrieval **A**ugmented **G**eneration = 검색으로 보강된 생성

ChatGPT 같은 LLM에게 질문하면, **학습 데이터에 있는 것만** 답한다.
우리 회사 내부 문서, 오늘 뉴스 같은 건 모른다.

**해결책:**

1. 질문과 관련된 문서를 먼저 "검색"한다
2. 검색 결과를 LLM에게 "참고자료"로 넘긴다
3. LLM이 그 참고자료를 기반으로 답변한다

> 비유: **오픈북 시험**. LLM이 혼자 기억으로 답하는 게 아니라, 관련 자료를 펼쳐놓고 답하는 것.

## 파이프라인 전체 흐름

```
[준비 단계 - 1회]
문서 → 쪼개기(청킹) → 숫자로 변환(임베딩) → 벡터DB에 저장

[질문 단계 - 매번]
질문 → 임베딩 → 벡터DB에서 유사한 청크 검색 → LLM에 질문+검색결과 전달 → 답변
```
